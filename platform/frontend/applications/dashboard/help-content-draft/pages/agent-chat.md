# Platform Agent

## Before you send a message

Replies often take **a few minutes** — stay on the page and wait until the request finishes; the chat updates when it completes.
**Do not** refresh while waiting — you may lose the current thread view.
There is no way to cancel a request once it is sent.

## What it can help with

- CVE overviews and **research summaries**.
- **Portfolio triage** — critical issues across projects.
- **Image-CVE advice** for a CVE on a specific component.
- **Web research** when fresher context is needed.

The agent reads platform data; it does not upload images or run scans for you.

## Web research

- **Search** finds links in chat; it does **not** save pages by itself.
- To **save a webpage** to research documents, name the **CVE id** in your message (e.g. *"For CVE-2024-1234, fetch the NVD page"* or *"Explore CVE-2024-1234 and save the vendor advisory"*).
- Saved pages appear under that CVE’s **Research documents** (source: **Agent lookup**).
- The CVE must already exist in the platform (from a scan or **Create CVE**).

## Example prompts

- *"Give me a research summary for CVE-2024-1234."*
- *"Which critical CVEs affect my projects?"*
- *"What should we do about CVE-2024-1234 on [component name]?"*
- *"For CVE-2024-1234, search the web and save the official advisory to research documents."*

## Saving changes

The agent usually shows a **draft** first. Saving a **research summary** or **advice** to the database needs your **confirmation** in a follow-up message.

## Threads and chat

- **New thread** — start a fresh conversation from the sidebar (or when none is selected).
- Select a thread to view messages; a **…** indicator shows while that thread is waiting for a reply.
- **Download** (under the message box) exports the current thread as markdown.
- Collapse the sidebar with **‹** if you need more room for messages.
